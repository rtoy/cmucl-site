The latest formal release of CMUCL (binaries and source) is available from the various download sites, in the release directory. For each supported platform there are two bzipped tar archives, one containing the base system, and the other (with extra in the name) containing additional files supporting CLX, Hemlock, Gray streams and so on.

The download sites are:

cmucl.cons.org:/pub/lisp/cmucl
The primary CMUCL machine, located in the United States. You can also access the same files using HTTP.
ftp.averillpark.net/cmucl
A complete mirror located in the United States.
ftp.linux.org.uk/pub/lisp/cmucl
A complete mirror located in England.
www.pmsf.de/pub/cmucl
A partial mirror of the releases and ports sections, which should provide good bandwidth across Europe and the USA.
www.pmsf.net/cmucl
The primary mirror and master rsync server, but only has limited outbound bandwidth.
ftp.cn.freebsd.org/pub/cmucl
A complete mirror, which should have reasonable bandwidth from Asia.

Binaries built from source more recent than the latest release are available from the download mirrors in the binaries directory. These binaries contain bugfixes and new features that are not in the release binaries, but have had less testing.

Binaries providing experimental features (such as long floats), or built with non-standard features or subsystems, are available in the experimental directory. These haven't had much testing.

Please read the CMUCL disclaimer for details of the absence of warranty traditional with free software.

Alternative binary distributions

CMUCL is available as a FreeBSD port (ports/lang/cmucl) for the x86, thanks to Martin Cracauer.

CMUCL is available as a .deb package for Debian GNU/Linux on x86, thanks to Peter Van Eynde. Note that the Debian binaries are not exactly equivalent to the cons.org binaries; they include a number of modifications. In particular, they are not binary-compatible with the cons.org binaries, so you cannot share FASL files between them. You can use Alien to convert the debian packages into RPMs to be installed on RedHat or SuSE installations: 

    # alien --to-rpm cmucl-version.deb
    # rpm -i cmucl-version.rpm

CMUCL is available in RPM format for Red Hat/x86, thanks for Miles Egan. The RPMs should be equivalent to the official release tarballs, but may be easier to install for some users.

CMUCL is available as a package for the Gentoo Linux distribution. The package is named cmucl-bin, and is in the dev-lisp category.

Anonymous CVS access

The CMUCL source code can also be obtained by anonymous CVS from anoncvs@anoncvs.cons.org:/home/anoncvs/CVS-cmucl. The anoncvs tree is synchronized daily with the main CVS tree, and can be accessed using a command such as 

   export CVSROOT=:pserver:anonymous@anoncvs.cons.org:/home/anoncvs/CVS-cmucl
   cvs login  (password is `anonymous')
   cvs co src

The anoncvs tree is available using anonymous rsync at rsync://cmucl.cons.org/CVS-cmucl. This is probably the most network-friendly way of keeping up to date with the current sources, since the rsync protocol is very efficient. Keeping your own local CVS tree also makes cvs operations much faster than when you contact the main repository. Sample usage: 

    rsync -avz rsync://cmucl.cons.org/CVS-cmucl /var/cvs/CVS-cmucl

You can also browse individual CVS files (these are not outchecked source files, but the RCS files!) at www2.cons.org. This is intended for mirroring the whole CVS tree using some web mirror tool (such as GNU wget). If you want the CVS tree as a whole, there is a daily tarfile of the CVS tree at www2.cons.org, using the filename cmucl-cvs-YYYYMMDD.tar.gz. Likewise, the file named cmucl-src-YYYYMMDD.tar.gz is a tarball of the sources, updated daily.

There are CVS tags for the latest releases, named RELEASE_18d, RELEASE_18c, RELEASE_18b and RELEASE_18a.

Thanks for using CMUCL!
